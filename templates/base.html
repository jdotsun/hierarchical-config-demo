How to Use the Configuration Management System
Understanding Scope Hierarchy

The system uses a hierarchical scope-based resolution to find the most appropriate configuration value. The scope hierarchy (from most local to most global):

  1. Account - Most specific, highest priority
  2. Model
  3. Model Family
  4. Model Provider
  5. Default - Least specific, lowest priority
Managing Configuration Items

Configuration items are the keys that you want to configure values for. Each item has a key, description, and value type.

  • To create a configuration item, use the "Create Configuration Item" form.
  • To delete a configuration item, click the delete button next to the item in the table.
Setting Configuration Values

Configuration values are assigned to specific scopes. When resolving a configuration value, the system will try to find the most local scope that matches the object's properties.

  • To set a configuration value, use the "Set Configuration Value" form.
  • When using a "default" scope type, you don't need to provide a scope value.
  • For other scope types, the scope value should match the corresponding property of the object.
Resolving Configuration Values

When you resolve a configuration value, the system will find the best match based on the object's properties and the scope hierarchy.

  • Select a configuration item to resolve.
  • Add properties to describe the object.
  • Click "Resolve" to find the most appropriate value.
